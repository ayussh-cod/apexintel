# Gennady Korotkevich — Actionable Tasks

> Derived from the strategy of Gennady Korotkevich in competitive programming.

## Phase 1 — Foundation (Week 1-2)
- [ ] Review fundamental data structures: arrays, linked lists, stacks, queues.
- [ ] Implement classic sorting algorithms: bubble, selection, insertion, merge, quick sort.
- [ ] Solve at least 2 easy problems daily on platforms like LeetCode or Codeforces.
- [ ] Understand basic time and space complexity analysis (Big O notation).
- [ ] Learn input/output methods for competitive programming in preferred language.
- [ ] Practice string manipulation and basic arithmetic problems.

## Phase 2 — Build (Week 3-6)
- [ ] Master graph traversal algorithms: BFS, DFS.
- [ ] Understand dynamic programming concepts with basic examples.
- [ ] Solve at least 3 medium problems daily focusing on specific topics.
- [ ] Participate in at least one virtual contest weekly on Codeforces or AtCoder.
- [ ] Study common number theory concepts relevant to competitive programming.
- [ ] Learn about binary search and its applications beyond simple search.

## Phase 3 — Compound (Month 2-3)
- [ ] Analyze past contest performance to identify weak areas (e.g., DP, geometry, specific data structures).
- [ ] Focus dedicated study sessions on identified weak areas, solving 5+ hard problems weekly in those areas.
- [ ] Review solutions of top competitors for unsolved problems in recent contests.
- [ ] Contribute to problem-setting or tutorial writing for a local programming club.
- [ ] Set a target Codeforces rating increase for the next 3 months (e.g., +200 points).
- [ ] Practice advanced data structures: segment trees, Fenwick trees, tries.

## Daily / Weekly Rituals
- [ ] Solve 2-3 new problems daily, alternating between easy and medium difficulty.
- [ ] Participate in one official Codeforces/AtCoder contest per week.
- [ ] Review and understand all problems from the weekly contest, including those not solved.
- [ ] Spend 30 minutes daily reviewing notes on algorithms and data structures.
