# Makoto Soejima — Actionable Tasks

> Derived from the strategy of Makoto Soejima in competitive programming.

## Phase 1 — Foundation (Week 1-2)
- [ ] Choose and master one primary programming language (e.g., C++, Java, Python) for competitive programming.
- [ ] Learn fundamental data structures: arrays, linked lists, stacks, queues.
- [ ] Understand basic algorithms: sorting (bubble, selection, insertion, merge, quick), searching (linear, binary).
- [ ] Practice solving at least 10 easy problems on a platform like Codeforces (Div3 A/B) or LeetCode Easy.
- [ ] Set up a competitive programming development environment (IDE, compiler, debugger).
- [ ] Familiarize yourself with standard library functions and common shortcuts for your chosen language.

## Phase 2 — Build (Week 3-6)
- [ ] Study intermediate data structures: trees (binary, BST, AVL, Red-Black), graphs, heaps, hash maps/sets.
- [ ] Learn core graph algorithms: DFS, BFS, Dijkstra's, Floyd-Warshall, Prim's, Kruskal's.
- [ ] Master dynamic programming fundamentals: knapsack, longest common subsequence, memoization vs. tabulation.
- [ ] Participate in at least one online contest per week (e.g., AtCoder Beginner Contest, Codeforces Div2).
- [ ] Analyze solutions to problems you couldn't solve during contests or practice sessions.
- [ ] Practice recognizing common algorithm patterns in problems.

## Phase 3 — Compound (Month 2-3)
- [ ] Explore advanced data structures: segment trees, Fenwick trees (BITs), Disjoint Set Union (DSU).
- [ ] Study advanced algorithms: number theory (GCD, LCM, primes), combinatorial math, string algorithms (KMP, Z-algorithm).
- [ ] Practice solving problems from past ICPC World Finals or regional contests.
- [ ] Deep dive into advanced problem-solving techniques: binary search on the answer, meet-in-the-middle, greedy approaches.
- [ ] Read and understand editorials for challenging problems, even if you solved them, to learn alternative approaches.
- [ ] Consistently review tricky problems and algorithms to reinforce understanding.

## Daily / Weekly Rituals
- [ ] Solve 2-3 new competitive programming problems daily.
- [ ] Participate in 1-2 online contests per week.
- [ ] Spend 30 minutes daily reviewing a difficult concept or a past problem.
- [ ] Read 1-2 competitive programming blog posts or tutorials weekly.
- [ ] Spend 1 hour weekly debugging and refining code for accuracy and efficiency.
