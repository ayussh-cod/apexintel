# Andrew He — Actionable Tasks

> Derived from the strategy of Andrew He in competitive programming.

## Phase 1 — Foundation (Week 1-2)
- [ ] Choose a primary online judge platform (e.g., Codeforces, LeetCode) and create an account.
- [ ] Learn basic data structures: arrays, linked lists, stacks, queues, hash tables.
- [ ] Practice fundamental algorithms: sorting (e.g., bubble, insertion, selection), searching (linear, binary).
- [ ] Solve at least 20 easy-level problems focusing on basic implementation.
- [ ] Understand time and space complexity (Big O notation) for common operations.

## Phase 2 — Build (Week 3-6)
- [ ] Study intermediate data structures: trees (binary, BST), heaps, graphs.
- [ ] Master core algorithms: dynamic programming (basic 1D problems), graph traversal (BFS, DFS).
- [ ] Participate in at least one short online contest per week.
- [ ] Solve at least 20 medium-level problems, applying newly learned techniques.
- [ ] Review solutions to problems you couldn't solve, understanding the optimal approach.

## Phase 3 — Compound (Month 2-3)
- [ ] Explore advanced topics: segment trees, Fenwick trees, number theory, flow algorithms.
- [ ] Tackle challenging dynamic programming problems (2D, state compression).
- [ ] Regularly participate in longer contests (e.g., Div2 B-D on Codeforces).
- [ ] Upsolve problems from contests, aiming to solve at least half of the unsolved ones.
- [ ] Contribute to an open-source project or create a personal competitive programming template library.

## Daily / Weekly Rituals
- [ ] Solve 1-2 new problems daily from a targeted topic.
- [ ] Review 1-2 previously solved problems or their optimal solutions weekly.
- [ ] Participate in at least one virtual contest or live contest each weekend.

## Metrics to Track
| Metric | Target | Frequency |
|--------|--------|-----------|
| Problems Solved (Easy) | 50 | Weekly |
| Problems Solved (Medium) | 30 | Weekly |
| Contest Rating Change | +50 | Bi-weekly |
