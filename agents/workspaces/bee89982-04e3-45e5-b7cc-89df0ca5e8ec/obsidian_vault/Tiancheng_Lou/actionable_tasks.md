# Tiancheng Lou — Actionable Tasks

> Derived from the strategy of Tiancheng Lou in competitive programming.

## Phase 1 — Foundation (Week 1-2)
- [ ] Understand basic data structures: arrays, linked lists, stacks, queues.
- [ ] Learn fundamental algorithms: sorting (bubble, insertion, merge, quick), searching (linear, binary).
- [ ] Grasp time and space complexity (Big O notation) for common operations.
- [ ] Solve 10-15 easy-level problems on platforms like LeetCode or Codeforces.
- [ ] Set up a competitive programming environment (IDE, compiler).
- [ ] Familiarize yourself with standard library functions in your chosen language (e.g., C++ STL).
- [ ] Practice basic input/output operations and error handling.

## Phase 2 — Build (Week 3-6)
- [ ] Master advanced data structures: trees (BST, AVL, Red-Black), heaps, hash maps, graphs.
- [ ] Learn core algorithmic paradigms: dynamic programming, greedy algorithms, divide and conquer.
- [ ] Practice graph algorithms: BFS, DFS, Dijkstra's, Floyd-Warshall, minimum spanning trees (Prim's, Kruskal's).
- [ ] Solve 10-15 medium-level problems, focusing on identifying correct algorithm types.
- [ ] Participate in at least two virtual competitive programming contests.
- [ ] Understand number theory basics: primality tests, modular arithmetic, GCD/LCM.
- [ ] Begin to optimize code for constant factors and edge cases.

## Phase 3 — Compound (Month 2-3)
- [ ] Solve 10-15 hard-level problems, including those requiring multiple algorithms.
- [ ] Participate in 3-5 live competitive programming contests (e.g., Codeforces Round, AtCoder).
- [ ] Analyze solutions for problems you couldn't solve during contests or practice.
- [ ] Learn advanced topics: segment trees, Fenwick trees, Suffix Arrays, String Algorithms.
- [ ] Practice competitive programming heuristics and common tricks (e.g., two-pointers, meet-in-the-middle).
- [ ] Review common mistakes and anti-patterns in your problem-solving approach.
- [ ] Contribute to an open-source competitive programming library or problem set.

## Daily / Weekly Rituals
- [ ] Solve at least one competitive programming problem daily.
- [ ] Participate in a virtual or live contest every week.
- [ ] Review and understand solutions to problems you failed to solve or solved inefficiently once a week.
- [ ] Read competitive programming blogs or tutorials for new strategies twice a week.
