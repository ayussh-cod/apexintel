# Kamil Debowski — Actionable Tasks

> Derived from the strategy of Kamil Debowski in competitive programming.

## Phase 1 — Foundation (Week 1-2)
- [ ] Understand basic programming language syntax and data types.
- [ ] Learn fundamental data structures: arrays, linked lists, stacks, queues.
- [ ] Master basic algorithms: sorting (bubble, selection, insertion), searching (linear, binary).
- [ ] Solve at least 30 easy problems on platforms like Codeforces or LeetCode.
- [ ] Set up a preferred competitive programming environment (IDE, custom templates).
- [ ] Study time and space complexity analysis (Big O notation).

## Phase 2 — Build (Week 3-6)
- [ ] Dive deep into advanced data structures: trees (BST, AVL), graphs, hash maps.
- [ ] Learn intermediate algorithms: dynamic programming, greedy algorithms, two-pointers.
- [ ] Solve at least 50 medium problems, applying learned concepts.
- [ ] Participate in 2-3 short competitive programming contests (e.g., Codeforces Div 3).
- [ ] Practice common problem-solving techniques like recursion and backtracking.
- [ ] Begin reviewing solutions to problems you couldn't solve within 30 minutes.

## Phase 3 — Compound (Month 2-3)
- [ ] Explore advanced graph algorithms: Dijkstra, Floyd-Warshall, MST, network flow.
- [ ] Study competitive mathematics: number theory, combinatorics, modular arithmetic.
- [ ] Solve at least 70 hard problems, focusing on optimizing solutions.
- [ ] Participate in 4-6 regular competitive programming contests (e.g., Codeforces Div 2).
- [ ] Develop a personal library of frequently used data structures and algorithms.
- [ ] Systematically analyze contest performance to identify weak areas and improve strategies.

## Daily / Weekly Rituals
- [ ] Solve 1-2 new competitive programming problems daily.
- [ ] Review 1-3 previously failed or difficult problems weekly.
- [ ] Participate in at least one online contest per week.
- [ ] Dedicate 30 minutes daily to reading algorithm tutorials or competitive programming blogs.
- [ ] Practice specific algorithm categories (e.g., DP, graph traversal) for 1 hour, twice a week.

## Metrics to Track
| Metric | Target | Frequency |
|--------|--------|-----------|
| Problems Solved (New) | 10 per week | Weekly |
| Contest Rating Improvement | +50 points | Monthly |
| Accuracy on First Submissions | 60% | Weekly |
| Time Spent Studying Algorithms | 10 hours | Weekly |
| Number of Algorithm Topics Covered | 2 new | Monthly |
